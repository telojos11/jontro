Picture-in-Picture Origin Spoofing via RTL Characters

Security UI Spoofing Chrome 143+ CWE-451

Attack Summary

When a website uses Right-to-Left (RTL) Unicode characters in its domain, the browser's Picture-in-Picture (PiP) window displays a visually misleading origin. Instead of showing the true root domain, the PiP UI appears to show trusted brands (apple.com, google.com), enabling origin impersonation and phishing.

3. Attack Scenario

  1. Attacker registers a domain with RTL characters in subdomains (e.g., xn--mgb.accounts.login.apple.com.xn--mgb.https.google.com.summa.sbs)
  2. Victim visits the malicious URL (e.g., from a phishing email or ad)
  3. The page automatically opens a PiP window playing a video or showing content
  4. The PiP window's origin label shows "apple.com" instead of the real root domain "summa.sbs"
  5. Victim trusts the PiP content believing it's from a legitimate source

4. How the Domain Spoof Works

Decoded domain:
ا.accounts.login.apple.com.ا.https.google.com.summa.sbs

Punycode (what DNS sees):
xn--mgb.accounts.login.apple.com.xn--mgb.https.google.com.summa.sbs

The Arabic Alef character (U+0627) is a strong RTL character.
It triggers the Unicode bidirectional algorithm to visually reorder text,
making summa.sbs appear as apple.com in the PiP UI.
