Chrome Fullscreen UI Spoofing POC

Security UI Spoofing CWE-451 Chrome Android/Desktop

Attack Summary

When a page enters fullscreen mode, the browser's URL bar and all security indicators are completely hidden. The page can then draw fake browser UI — a spoofed URL bar showing a trusted domain (google.com) with a lock icon. The victim has no way to verify the real URL without knowing the fullscreen exit gesture.

How It Works

1. Social engineering — page shows a fake prompt: "Google needs to open a secure session"
2. User taps "Continue" → page enters fullscreen via requestFullscreen()
3. Real URL bar disappears — browser chrome is hidden
4. Page draws fake Chrome UI with spoofed URL + lock icon
5. Victim sees "google.com" in the fake URL bar — trusts the page completely
6. Phishing form / download / permission prompt follows

Why This Works on Android Chrome

• Fullscreen hides: URL bar, status bar, navigation buttons
• Exit gesture (swipe from top) is not intuitive for most users
• Fake browser UI is indistinguishable from real Chrome UI
• Combined with a Google-lookalike page = perfect phishing
• No patch exists — hiding UI is the intended behavior of fullscreen API

Variants Demonstrated

Fake Chrome Browser: Complete fake Android Chrome UI (status bar + URL bar + tabs)
Google Search Spoof: Fake search results + download prompt showing "from google.com"
Google Sign-In: Full Google login flow phishing with fake accounts.google.com URL
