============================================================
SECURITY VULNERABILITY REPORT
============================================================

ONE LINE SUMMARY (100 character limit):
------------------------------------------------------------
Fullscreen API enables UI spoofing: fake Chrome URL bar shows trusted domain while real origin is hidden.

------------------------------------------------------------
STEPS TO REPRODUCE (5000 character limit):
------------------------------------------------------------
1. Host the attached spoof.html on a web server accessible from Android Chrome.

2. Open the page on Android Chrome (tested on Chrome 148.0.7778.178).

3. Observe the page displays a Google-lookalike homepage with search box and "Google Search" / "I'm Feeling Lucky" buttons.

4. Tap the search box or either button.

5. The page calls requestFullscreen(). The browser enters fullscreen mode, completely hiding the real URL bar, status bar, and all browser security indicators.

6. After entering fullscreen, the page draws a fake Android Chrome browser UI consisting of:
   - Fake status bar (time, signal, battery)
   - Fake URL bar with green lock icon showing "www.google.com"
   - Fake tab strip showing "Google" and "New Tab" tabs

7. The page displays fake Google search results, including a malicious result labeled "Google Account Security Verification - Required" that appears to link to "accounts.google.com".

8. Tap the malicious search result. The fake URL bar updates to show "accounts.google.com".

9. A fake Google Sign-in page appears, identical to the real Google login page, requesting email/phone and password.

10. Enter any email and password. The fake URL bar updates to "myaccount.google.com" and a success message appears: "Verification Complete - Your account is secure."

11. After 2 seconds, the page redirects to the real Google Security Checkup page (myaccount.google.com/security-checkup).

12. Throughout steps 5-10, the victim cannot see the real URL (telojos11.github.io) because:
    - Fullscreen mode hides the real browser URL bar
    - The page draws a fake Chrome UI showing google.com
    - No download notification or system prompt reveals the real origin
    - Exiting fullscreen on Android requires swiping from top edge, which is not intuitive for most users

13. To verify the real origin: exit fullscreen by swiping from the top edge of the screen, or check the URL after the redirect completes.

------------------------------------------------------------
PROBLEM DESCRIPTION (5000 character limit):
------------------------------------------------------------
The HTML Fullscreen API (Element.requestFullscreen()) allows web pages to hide all browser security indicators including the URL bar, status bar, and navigation buttons. While in fullscreen mode, the page can render arbitrary fake browser UI that is visually indistinguishable from the real Chrome browser interface.

This vulnerability exploits the fullscreen API to conduct a complete UI spoofing attack:

1. FAKE BROWSER CHROME: The page draws a fake Android Chrome browser interface including status bar (time, signal strength, battery), URL bar with a green lock icon showing "www.google.com", and tab strip. This fake UI is visually identical to the real Chrome browser chrome on Android.

2. ABSENT SECURITY INDICATORS: The real browser URL bar, lock icon, and all security indicators are completely hidden by fullscreen mode. The victim has no visual cue that the displayed URL bar is fake.

3. CONVINCING SOCIAL ENGINEERING: The page mimics Google Search results with a fake security alert claiming "Unusual activity detected on your account." The fake result appears to link to "accounts.google.com" with a green lock icon, making it indistinguishable from a legitimate Google security notice.

4. CREDENTIAL HARVESTING: The fake Google Sign-in page captures the victim's email/phone and password. The form is pixel-identical to the real Google login page, with matching fonts, colors, layout, and interactive behavior (email step → password step → verification animation → success page).

5. CLEAN EXIT: After capturing credentials, the page redirects to the real Google Security Checkup page (myaccount.google.com/security-checkup). The victim sees a legitimate Google page and believes the security verification was successful.

6. NO ORIGIN LEAKAGE: Unlike download-based attacks, this credential phishing approach does not trigger any browser download notification, system dialog, or permission prompt that would reveal the real origin (telojos11.github.io). The entire attack occurs silently within fullscreen mode.

7. ANDROID-SPECIFIC IMPACT: On Android Chrome, exiting fullscreen requires the user to swipe from the top edge of the screen. This gesture is not intuitive and many users do not know it. Additionally, fullscreen on Android hides the system status bar, making it harder to distinguish the fake status bar from the real one.

The fundamental issue is that the Fullscreen API was designed for media consumption (videos, presentations, games) where hiding browser UI is expected and safe. However, it provides no mechanism for the browser to indicate that the displayed UI elements are page-rendered rather than browser-rendered. The page has unrestricted ability to draw any UI, including fake browser chrome, while in fullscreen mode.

Suggested mitigations:
- Display a persistent, non-spoofable browser indicator while in fullscreen mode (e.g., a subtle overlay showing the real origin)
- Limit the Fullscreen API to user-initiated gestures with a confirmation dialog showing the real origin
- Show a transient browser UI element when exiting fullscreen to re-establish trust
- Add a browser-managed "exit fullscreen" indicator that pages cannot draw over or hide

This vulnerability affects all Chromium-based browsers on Android and desktop platforms.
