Download Origin Spoofing

Security UI Spoofing Chrome Android/Desktop Data URI Abuse

Attack Summary

By crafting a data: URI that embeds a fake hostname (data:https://google.com/...[spaces]...application/x-msdownload;base64,...), Chrome's download prompt displays google.com as the origin instead of the actual hosting page. The real file type and origin are hidden behind many spaces, making the download appear legitimate.

Impact: Victim sees "google.com" in the download prompt and trusts the downloaded file (APK, EXE, DMG, etc.) is from Google.

1. Core Attack - Download Origin Spoof

Click any button. Each triggers a download with a spoofed origin shown in Chrome's download bar/prompt.

2. Advanced Spoof Variations

Different techniques to make the spoof more convincing on Android.

3. Android-Specific Bypass Techniques

Techniques that exploit Android Chrome's download UI behavior.

4. How It Works

data:https://google.com/[215+ spaces]application/x-msdownload;base64,[payload]

│     │                            │                                   │
│     └─ Fake origin               └─ Hidden MIME type                 └─ Encoded file content
│         (visible in UI)               (pushed out of view)
└─ data: scheme

Chrome's download manager extracts the first URL-like pattern from
the data URI and displays it as the download origin.
The spaces push the real MIME type indicator far to the right,
completely out of the download prompt's visible area.
